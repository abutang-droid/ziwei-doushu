# 紫微斗数应用改造开发文档

## 1. 改造背景与目标

当前紫微斗数应用基于 Next.js 14 架构，已具备单人排盘、合盘（部分功能）以及基于 DeepSeek API 的 AI 解读能力。为了更好地适应作为 WordPress 嵌入式模块的需求，并提升用户体验，计划进行以下四项核心改造：

1. **界面重构**：修改现有界面，使其符合传统的“八字计算器”风格。
2. **功能整合**：将目前分离的单人解盘与合盘功能整合到同一个界面中。
3. **嵌入式优化**：去除应用的页头（Header）和页尾（Footer），使其成为纯粹的嵌入式组件。
4. **AI 接口全面启用**：确保 AI 接口在整合后的界面中能够稳定调用。

经过对项目源码的深度评估，**这四项改造在技术上完全可行，且现有架构能够很好地支持这些变更。**

---

## 2. 需求评估与可行性分析

### 2.1 界面修改：符合八字计算器风格

**现状分析**：
当前应用采用现代深色主题（Dark Theme），使用大量卡片式布局和发光效果（如 `app/chart/page.tsx` 和 `globals.css` 中的定义）。

**改造方案**：
*   **颜色与主题**：需要调整 `globals.css` 中的 CSS 变量，将背景色、文字颜色调整为经典的“八字计算器”风格（通常是浅色背景、宣纸质感或简洁的白底黑字/红字）。
*   **布局调整**：去除不必要的阴影和圆角，将卡片式布局改为更紧凑的表格或网格布局，突出核心命盘数据。
*   **美学重构**：根据设计规范，不使用背景图，从美学角度进行重构，保持界面干净清爽。

**可行性**：高。项目使用了 Tailwind CSS 和 CSS 变量，只需修改主题配置和少量组件样式即可实现全局风格切换。

### 2.2 整合单人解盘与合盘功能

**现状分析**：
目前单人排盘入口在 `app/page.tsx`，展示在 `app/chart/page.tsx`；合盘（合婚）功能独立在 `app/heming/page.tsx` 中。两者使用了不同的状态管理和页面路由。

**改造方案**：
*   **统一入口表单**：重构 `components/BirthForm.tsx`，在输入区域下方增加“单人/双人”模式切换按钮。
*   **动态选项卡**：在双人模式下，不采用左右并列布局（以适应嵌入式窄屏），而是使用动态选项卡（Tab）让用户在“命主”和“对方”之间切换输入。
*   **统一展示界面**：在排盘结果页，如果是双人模式，增加一个“合盘分析”的独立面板，同时可以切换查看两人的单盘。

**可行性**：高。React 组件的复用性很好，只需在顶层组件中增加模式状态（`isCoupleMode`），并根据状态渲染相应的表单和结果视图。

### 2.3 嵌入式应用改造（去头去尾）

**现状分析**：
在 `app/chart/page.tsx` 和其他页面中，包含了 `<header>`（返回按钮、标题）和 `<TimeNav>` 等导航元素。

**改造方案**：
*   **移除全局导航**：清理 `app/layout.tsx` 或具体页面中的全局 Header 和 Footer。
*   **优化内部导航**：对于返回上一步或重新排盘的操作，改用组件内部的按钮，而不是依赖页面级的路由跳转。
*   **自适应高度**：由于作为 iframe 嵌入 WordPress，需要确保应用内容能够自适应高度，或者使用 `postMessage` 向父窗口传递高度信息，避免出现双重滚动条。

**可行性**：极高。只需删除或隐藏相应的 JSX 节点，并调整外层容器的 `minHeight` 和 `padding`。

### 2.4 启用 AI 接口

**现状分析**：
项目中已包含 `components/ChatPanel.tsx`，通过 `/api/interpret` 路由调用 DeepSeek API。目前在单人盘中有调用，但在合盘页面中可能不够完善。

**改造方案**：
*   **统一 AI 调用逻辑**：确保整合后的界面能够将单人命盘数据或双人合盘数据正确序列化并发送给 `/api/interpret`。
*   **Prompt 优化**：针对合盘场景，需要在 API 路由中增加专门的 Prompt 模板，指导 AI 进行两人的缘分、性格匹配度分析。
*   **环境变量配置**：确保 Vercel 部署环境中已正确配置 `DEEPSEEK_API_KEY`。

**可行性**：高。底层 API 路由和流式输出（Streaming）组件已存在，只需根据新的数据结构调整入参和 Prompt 即可。

## 3. 开发实施计划

整个改造过程预计可以分为三个阶段进行：

### 阶段一：结构整合与嵌入式改造
1.  **清理页面头尾**：移除所有页面中的 `<header>` 和不必要的导航栏，将外层容器调整为适合 iframe 嵌入的样式。
2.  **重构输入表单**：在 `BirthForm` 中实现单/双人模式切换，使用选项卡管理双人输入。
3.  **路由合并**：将 `heming` 页面逻辑合并到主流程中，实现单一入口。

### 阶段二：风格重塑（八字计算器风格）
1.  **修改 CSS 变量**：在 `globals.css` 中定义新的浅色系/经典命理风格主题。
2.  **重写组件样式**：调整 `PalaceCell`（宫位格子）、`ChartBoard`（命盘主板）的边框、颜色和字体，使其看起来更像传统的排盘纸。
3.  **移除冗余装饰**：去除发光、渐变等现代 Web 效果，追求简洁美学。

### 阶段三：AI 功能联调
1.  **升级 ChatPanel**：使其能够接收单人或双人命盘数据。
2.  **后端 API 适配**：修改 `/api/interpret/route.ts`，增加对合盘分析的支持。
3.  **全面测试**：在本地环境和 Vercel 预览环境中测试完整流程。
